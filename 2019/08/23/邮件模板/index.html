基本内容编写

建议保持的代码结构

删除多余的标签，尽量避免使用除了 table、html、meta、head之外的标签，div在细节处理的时候可以适当使用，不使用 p 标签，整体保证 table 布局。

HTML编码使用 utf-8

使用table布局

在这里，我遇到非常多坑。

在查阅众多资料时，大家都不约而同提到说使用table布局，但理由是利用table本身的一些属性进行布局，可我查阅MDN官方文档时，却发现很多table里很多属性在现在已经不再支持了，于是我采用了<div></div> ，这在后期遇到了非常多的问题，这里就先暂且不提。

况且，因为不能使用HTML5语法，并且客户端对CSS的支持情况不尽相同，所以我们整体布局要使用table布局，且必须使用table布局，就不要使用div p等标签了。

创建body和主体表格

这里建议表格宽度为600-800px，因为这个宽度会使得你的邮件在桌面和客户端以及大多数的屏幕分辨率下显示的最舒服，且不会超过客户端的显示宽度。

这里先在最外层设置的<table></table>是用来设置背景的，border=”1” 只是为了方便开发，在发布时应设置为0。

值得注意的是，设置border-collapse的属性值为collapse。如果我们不这样做，新版本的Outlook将会给我们的表格与边框之间添加一个小的空间距。

创建结构和头部

看邮件整体设计，分为几个逻辑部分就设置几行

样式编写

如果一个属性存在于HTML中，绝对使用标签属性而不使用CSS样式。

充分利用table私有属性

充分利用width, height, bgcolor, background, align, valign等。

不需要考虑DOM节点的精简和结构的优化，不必吝啬使用空的表格元素来占据空间，只要完成设计效果即可。

使用内联样式

有一部分邮件客户端只支持内联样式，并且会添加一些奇奇怪怪的类名，进而导致我们的样式失效。

并且，不能使用<style type="text/css"></style>来处理主要样式，所有的Web邮件系统都会过滤该标签。

但对于习惯写外联样式的程序员来说，这会导致在编写代码时非常不顺手，开发和维护的成本都太高了。

所以在这边我推荐一个线上的内联样式编译工具CSS Inliner Tool。通过这个，就可以将<style></style>中的样式，编译到对应的标签内。

不用margin

部分客户端会有自己的默认样式，当我们在客户端进行查看发现内容之间存在的空隙时，往往会忽视这只是某些客户端的默认样式，从而找不到问题所在。

  • 为所有元素都应用margin: 0;这个属性来排除干扰。
  • 采用padding代替margin来增加空隙。

背景

不允许使用

应使用

因为CSS背景图片是一种会影响页面渲染速度的定义，因此大多数Web邮件系统会过滤。

  • 不建议使用<img></img> ，因为会有4px的空白部分。
  • 背景颜色可以采用table的自带属性 bgcolor
  • 但值得注意的是，outlook对背景图片不识别
  • 在有背景图的地方设置background-color / bgcolor 以防背景图不显示时可以较美观的显示

图片

  • 所有图片都定好具体的宽高，使用精确像素值
  • 去掉图片之间的4px空白缝隙 必须加上的 top 也可以换成bottom，但是设置要求一致
  • 不要有太多图片，会被定义成垃圾邮件，能切图直接上手切，能拼接直接拼。在查资料的时候有看到：整页图片控制在8张以内，每张图片最大不能超过15KB。但经自己实际测试，超过八张其实也没事，但还是不建议太多图片，图片大小不超过100KB比较好，太大太多都可能被归类为垃圾邮件。
  • 图片名称不能含有ad字符，否则图片上传后会显示成“被过滤广告”
  • 如果嵌套的图片太长，会导致显示不正常，可将图片裁切成几张较短的小图，再进行布局
  • 不使用flash、Java、JavaScript、frames、iframe、activeX以及DHTML，如果页面中的图片一定要动态的，请将flash文件转换成gif动画使用，但在outlook2007里，gif将不能正常显示，因为outlook2007限制gif动画。

文字

这里也踩了很多坑，部分客户端没办法加上字体颜色，在查阅众多资料后，找到了最不会出错的一种方法。

  • 不使用类似如下敏感及带促销类的文字：

​ 如果一定需要，请把敏感字做成图片。

  • font-family 只支持系统字体，不支持自定义字体，也不支持 font 简写
  • 字体颜色尽可能也不要使用简写（使用完整的6个完整的十六进制编码，使用三个简写字符不总是有效）
  • 对于加粗字体，可以使用b标签而不要使用font-weight
  • font-family属性不能为空，否则会被QQ屏蔽为垃圾邮件。

链接

  • 任何链接都写成绝对地址。
  • 链接地址的长度不能超过255个字符，会导致无法追踪或链接错误。
  • 为避免用户收到的邮件图片无法浏览，请制作一份和邮件内容一样的web页面，然后在邮件顶部注明：“如果您无法查看邮件内容，请点击这里”，链接到拥有同样内容的web页面。
  • 链接地址的长度不能超过255个字符，会导致无法追踪或链接错误。

垂直水平居中

​ 在Web邮件中，会自动为你在最外层生成一个<div style=”width:apx; margin:auto”> 的元素，不要自己 在邮件模板最外层画蛇添足。

  • 使用table自带属性 align=”center”
  • 值得注意的是：在yahoo邮箱中，在table里设定align=”center”无法居中， 需要内联style=“margin：0 auto，width：XX”

各种间距

  • 若希望邮件模板内侧边或者上下有空白间距，不要用 padding，必须得用标准的 td 来设定空白间距，否则会导致各个邮箱解析不同。
  • 或给（空白的） td 设置高度来控制垂直间距（占位）。
  • 给table 设置空白边距时，不要用 padding、marin ，应使用border-top等设置

table布局Tips

  • 尽量不在<tr></tr>元素上定义CSS样式，请将样式尽量定义在<td></td>元素上。

  • 不能使用伪类和伪元素以及高级选择符。

  • 可以使用<div></div>来实现细节的、具有典型块级元素的布局样式。而不能使用<p></p>，因为我们不容易清除<p></p>在不同浏览器的默认样式。但整体仍要保证table和元素样式来完成所有基本样式和布局。

  • 注意table不会继承font，在每个元素上都定义字体属性和颜色。

  • 永远记住要使用完整的6个完整的十六进制编码，如果使用三个简写字符，不总是有效

  • p 标签的 width 是不认识的，只有设置了宽度，内容才会在宽度限制处换行

  • 如果 td 里包含一个 table，那么定义在子 table 里的标签的样式只会正确显示定义在子 table 里的样式，在父 table 的样式是不能正确显示的。注意：如果在 table 里有 a 标签的话，则 a 标签中的内容的颜色样式要定义在 a 标签内，定义在父 td 里或所在的 table 里都是无效的。

不做响应式

没有必要，个人觉得意义不大，徒增烦恼。

采用固定的宽度，让各屏幕显示效果一致，手机端查看的邮件时候只需要适当地缩放就可以了。只要设计合理，字体大小合适，就没有什么大问题。

不同邮箱注意事项

outlook

  • 对背景图片不识别
  • .在里设置 margin 是无效的
  • 如果要使用<P></p>标签要考虑到本身自带的上下行之间的行高
  • 内联元素的 padding 、vertical-align无法被识别，可在td中设定
  • 标签不能被识别，如果要定义几个 td 相同的宽度，可以把具体的宽度写在对应的 td 里
  • 设置 table 的border-collapse的属性值为collapse。否则，新版本的Outlook将会给我们的表格与边框之间添加一个小的空间距。

fox mail

  • 所有p标签的 margin为 0； 使用 p 标签时需要设置margin
  • 如果 td 里有 p 标签，则 line-height 也必须在 p 中定义。
  • 无论是 td 还是 p，如果有超链接，则都必须在 a 标签里定义 line-height；如果只是在 td 或者 p 里面定义 line-height 的话，将无法识别 a 里面的行高。

yahoo

  • 需在块级元素里定义 line-height
  • table里设定align=”center”无法居中， 需要内联style=“margin：0 auto，width：XX”

line-height的一些小Tips

之前一直不明白

在这样的代码结构上，line-height 加在 div 上的和直接加在 span 上垂直间距为什么是不一样的？？？

原因是在span标签前面有一个隐藏的空白字符，且这个字符是继承父元素的字体大小，默认为14px ，因此在父元素 div 上加 line-height ，实际上是加在这个空白字符上的，line-height则是以这个空白字符的字体大小来变化的。

但当我们直接在 span 上加 line-height 的话，line-height就是直接依据这个 span 标签的实际字体大小来变化的。

因此，当 span 的字体大小不是14px时，作用在两者上的line-height 形成的垂直间距就会有所不同。
